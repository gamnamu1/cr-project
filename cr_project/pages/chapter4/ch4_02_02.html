웹앱 페이지와 메뉴 구성

1. 웹앱 정보구조(IA, Information Architecture) 설계

CR 플랫폼은 뉴스 기사의 품질을 정량적으로 평가하고, 시민들이 언론을 감시하고 견제할 수 있도록 지원하는 것을 목적으로 합니다. 아래는 이러한 목적을 달성하기 위한 사용자 중심의 정보구조입니다.

핵심 사용자 시나리오

  1. 발견 : 사용자가 의심스러운/문제적인 기사를 발견
  1. 평가 : 플랫폼에 접속하여 기사 URL 또는 텍스트 입력
  1. 분석 : AI 기반 다차원 매트릭스 평가 결과 확인
  1. 학습 : 평가 결과를 통해 기사의 문제점 이해
  1. 공유 : 평가 결과를 SNS나 이메일로 공유
  1. 토론 : 커뮤니티에서 다른 사용자들과 논의(선택 사항, 토론 공간을 플랫폼 내부에 둘지, 외부에 별도 커뮤니티를 활용할지 협의 후 결정)

정보구조도 

홈 랜딩페이지
├── 기사 평가 (메인 기능)
│   ├── URL 입력
│   ├── 텍스트 직접 입력
│   ├── 파일 업로드
│   └── 진행 상태 투명 표시
├── 평가 결과 (메인 기능)
│   ├── 종합 품질 점수
│   ├── 8개 차원별 레이더 차트
│   ├── 기사 하이라이트 (문제 표현 색상 코드화)
│   ├── 개선 제안
│   └── 결과 공유 도구
├── 커뮤니티(내부/외부 선택사항)
│   ├── 최근 평가된 기사
│   ├── 토론 게시판 (주제/언론사별)
│   ├── 문제적 보도 사례 아카이브
│   └── 집단 기획 분석 (이슈별 집중 모니터링)
├── 미디어 리터러시 센터(내부/외부 선택사항)
│   ├── 기사 비판적 읽기 가이드
│   ├── 문제적 보도 유형 설명
│   ├── 실전 사례 분석
│   └── 인터랙티브 교육 자료
├── 플랫폼 네트워크
│   ├── 연계 미디어 비평 플랫폼 소개
│   ├── 자매 플랫폼 개발 지원 자료
│   ├── API 문서 및 개발자 리소스
│   └── 확장 프로그램 다운로드
└── 프로젝트 정보
    ├── CR 프로젝트 소개
    ├── 평가 방법론 설명
    └── FAQ

2. 페이지 구성 및 메뉴 설계

주요 메뉴 구성

  • 주 메뉴 : 기사 평가 | 커뮤니티 | 미디어 리터러시 | 플랫폼 네트워크 | 프로젝트 소개
  • 보조 메뉴 : 다크모드 전환

주요 페이지 구성 요소

1) 랜딩페이지(홈)

  • 헤더 : 로고, 주/보조 메뉴, 3클릭 접근성 보장
  • 히어로 섹션 : 슬로건, 기사 URL 입력 필드(메인 CTA)
  • 실시간 통계 카운터 : 평가된 기사 수, 참여 사용자 수, 발견된 패턴 수
  • 가치 제안 : 핵심 가치 아이콘과 간결한 설명
  • 사용 방법 : 3단계 애니메이션 프로세스
  • 최근 평가 샘플 : 인기 있는 최근 평가 결과 카드형 표시
  • 언론사 협력 네트워크 맵 : 공간배치형 인터랙션
  • 신뢰도 배지 섹션 : 학술 인증 및 협력 기관 로고
  • 푸터 : 저작권, 소셜 미디어, 이용약관, 확장 플랫폼 링크

2) 기사 평가 페이지

  • 입력 방식 탭 : URL/텍스트/파일 업로드
  • 프로그레스 바 : 단계별 진행 상황 실시간 표시
  • 자동 저장 시스템 : 모바일 사용자 경험 최적화
  • 분석 시작 버튼 : 눈에 띄는 CTA 버튼

3) 평가 결과 페이지

  • 상단 요약 : 기사 정보, 종합 품질 점수 (색상 그래디언트 적용)
  • 레이더 차트 : 8개 차원 시각화
  • 기사 하이라이트 : 문제 패턴에 컬러 코드 적용, 마우스 오버 시 설명
  • 차원별 상세 분석 : 각 차원의 세부 점수와 근거 설명
  • 관련 규범 연결 : 발견된 문제와 언론 윤리 규범 연결 설명
  • 비교 분석 기능 : 동일 주제/언론사 기사와 비교
  • 공유 도구 : PDF 다운로드, SNS 공유, 이메일 공유, 임베드 코드

4) 플랫폼 네트워크 페이지

  • 미디어 비평 생태계 지도 : 다양한 비평 플랫폼 시각화
  • 자매 플랫폼 개발 가이드 : 기술 인프라 공유 방법
  • API 문서 : 개발자를 위한 기술 문서
  • 오픈소스 프로젝트 : 코드 저장소 및 기여 방법
  • 확장 프로그램 : 브라우저별 확장 프로그램 다운로드
  • 대시보드 : 언론사별/카테고리별 품질 추이 그래프

3. 랜딩페이지 중요 요소(우선순위별)

최우선 요소

  1. 기사 URL 입력 필드와 분석 시작 버튼
    • 3클릭 내 평가 완료 보장
    • 모바일 화면에서도 60% 이상 영역 차지
  1. 프로젝트 슬로건과 핵심 가치 설명
    • "함께 읽고, 함께 바꾸는 저널리즘"
    • 시민 주도 언론 개혁의 가치 명확히 전달
  1. 사용 방법 애니메이션
    • 4단계 애니메이션 (URL 입력 → AI 분석 → 사용자 평가 → 결과 공유)
    • 짧지만 명확한 설명으로 진입 장벽 제거

중요 요소

  1. 평가 시스템 신뢰도 표시
    • 8개 평가 차원 다이어그램
    • 학술 인증 배지 및 파트너 로고
  1. 샘플 평가 결과
    • 실제 분석 사례로 시스템 효과 입증
    • 색상 코드화된 신뢰도 점수
  1. 커뮤니티 활동 지표
    • 실시간 평가 현황 카운터
    • 최근 활발한 토론 주제 미리보기
  1. 미디어 비평 생태계 연결 지도
    • 다양한 미디어 비평 플랫폼 네트워크 시각화
    • 확장 가능성 강조

선택적 요소

  1. 브라우저 확장 프로그램 다운로드
    • 일상적 뉴스 소비 과정에 통합 가능성 강조
  1. 관리자 대시보드 미리보기
    • 시스템 투명성 강화를 위한 모니터링 도구 소개
  1. 사용자 평가 영상
    • 실제 평가 과정 15초 클립으로 신뢰 구축
  1. 비영리 운영 모델 설명
    • 투명한 재정 구조와 독립성 강조

4. 차별화 요소

  1. 플랫폼 생태계 확장 비전
    • 다양한 미디어 비평 플랫폼으로 확장되는 네트워크 구축
    • API 및 개발자 도구를 통한 생태계 확장 지원
  1. 하이브리드 분석 시스템
    • AI 알고리즘 + 시민 참여형 평가의 결합
    • 기술적 분석과 인간의 판단을 조화시킨 평가 모델
  1. 비영리 운영 투명성
    • 운영 비용, 의사결정 과정, 발전 방향의 투명한 공개
    • 시민 주도성 강화를 위한 거버넌스 구조
  1. 교육 - 평가 - 토론 - 행동의 선순환
    • 미디어 리터러시 향상
    • 기사 품질 평가 참여
    • 커뮤니티 토론 활성화
    • 언론 개혁 행동으로 연결

선택적 요소

  1. 소셜 미디어 공유 버튼
    • 이유: 플랫폼 확산과 사용자 네트워크 활용
    • 구현: 주요 소셜 미디어 플랫폼 공유 버튼, 미리 작성된 공유 문구
  1. 브라우저 확장 프로그램 다운로드
    • 이유: 일상적인 뉴스 소비 과정에 쉽게 통합 가능함을 보여줌
    • 구현: 주요 브라우저용 확장 프로그램 다운로드 링크 및 간단한 설명

이 설계안은 CR 프로젝트의 핵심 가치인 "시민 주도 언론 감시"와 "미디어 리터러시 향상"을 효과적으로 구현하면서, 사용자가 쉽게 참여하고 지속적으로 활용할 수 있는 플랫폼을 목표로 합니다. 기사 평가라는 핵심 기능을 중심으로, 평가 결과의 공유와 커뮤니티 토론을 통해 집단적 언론 개혁의 기반을 마련하는 구조입니다.
