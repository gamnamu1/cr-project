3-4. 사용자 경험 시나리오

구체적 사용자 경험 시나리오 (URL 입력 기준)

  1. 사용자가 웹앱에 접속하여 메인 페이지에서 "URL로 분석하기" 선택
  1. 분석할 뉴스 기사의 URL을 입력 필드에 붙여넣고 "분석 시작" 버튼 클릭
  1. 시스템이 로딩 애니메이션과 함께 다음 단계 진행:
    • "URL 확인 중..." (3초)
    • "기사 내용 추출 중..." (5초)
    • "기사 유형 분석 중..." (3초)
    • "품질 평가 진행 중..." (10초)
  1. 기사가 분석 가능한 유형(정치/사회 분야 스트레이트 뉴스)으로 확인되면:
    • 화면 상단에 종합 품질 점수(78/100점) 표시
    • 8개 평가 차원을 레이더 차트로 시각화
    • 기사 원문 표시 (문제 표현에 하이라이트 적용)
  1. 사용자가 하이라이트된 문장에 마우스 오버하면:
    • 툴팁으로 문제점 상세 설명 표시
    • 예: "무주체 피동형 표현('알려졌다')은 정보 출처를 불명확하게 만들어 투명성과 진실성을 저해합니다."
    • 개선 제안: "~회사 대변인이 밝혔다" 또는 "~보도자료에 따르면" 등으로 출처를 명확히 하는 것이 좋습니다.
  1. 상세 분석 탭에서:
    • 각 평가 차원별 세부 점수 및 근거 확인
    • 문제 패턴 발견 빈도 및 심각도 확인
    • 주요 개선 포인트 5가지 제시
  1. "PDF 리포트 다운로드" 버튼 클릭 시:
    • 분석 결과가 포함된 PDF 파일 생성 및 다운로드
    • 기사 원문, 평가 점수, 개선 제안이 포함된 전문 보고서
  1. "결과 공유하기" 버튼 클릭 시:
    • 고유 URL이 생성되어 클립보드에 복사
    • 소셜 미디어 공유 옵션 표시
  1. "커뮤니티 토론" 버튼 클릭 시:
    • Discord 서버 연결 및 해당 분석 결과 전용 채널로 이동
    • 다른 사용자들과 해당 기사의 품질에 대해 토론 가능
  1. 분석 완료 후:
    • "다른 기사 분석하기" 버튼으로 새로운 분석 시작 가능
    • 로그인한 사용자는 "분석 히스토리"에서 이전 분석 결과 확인 가능

이 최종 서비스 흐름도는 포괄적인 기능성과 사용자 경험을 기반으로, 에러 처리와 예외 상황 관리를 체계적으로 통합하여 더욱 안정적이고 사용자 친화적인 시스템을 구현합니다.
